═══════════════════════════════════════════════════
[Phase 1] Intent Parsing (의도 분석)
═══════════════════════════════════════════════════

사용자 요청: "raw data 엑셀에서 데이터를 읽어서 양식에 붙여넣고,
일/주/월별 불량률을 산출한 후 메신저로 전파해줘"

하위 목표 분해:
  1) raw_data.xlsx에서 생산 데이터 추출
  2) 양식(defect_rate_template.xlsx)의 '데이터입력' 시트에 삽입
  3) 수식에 의한 일/주/월별 불량률 자동 계산 (양식 내장)
  4) 계산 결과를 메신저로 전파

암묵적 요구사항:
  - "붙여넣어"라는 표현 → 기존 데이터를 먼저 클리어해야 함
  - 양식에 수식이 있으므로 RECALCULATE 필요
  - 메신저 발송은 외부 행위이므로 require_confirm 설정 필요

필요 능력:
  - 파일 I/O (OPEN_FILE, SAVE_FILE)
  - 엑셀 읽기/쓰기 (READ_COLUMNS, WRITE_DATA, CLEAR_RANGE)
  - 수식 재계산 (RECALCULATE)
  - 메시지 구성 (FORMAT_MESSAGE)
  - 메신저 API (SEND_MESSENGER)

═══════════════════════════════════════════════════
[Phase 2] Context Acquisition (맥락 수집)
═══════════════════════════════════════════════════

양식 구조 분석 (defect_rate_template.xlsx):

  시트 1: '데이터입력'
    - 역할: raw 데이터 삽입 영역
    - B1:F1 = 헤더 (날짜, 라인명, 생산수량, 불량수량, 불량유형)
    - B3:F10000 = 데이터 입력 영역 (B3부터 시작)
    - 수식 없음 (순수 데이터 영역)

  시트 2: '일별현황'
    - 역할: 일별 불량률 자동 계산 결과
    - A열: 날짜, B열: 라인명, ..., G열: 불량률(%)
    - G열에 수식 있음: =E열/D열*100 (불량수량/생산수량*100)
    - A2:G10 범위에 결과 출력

  시트 3: '주별현황'
    - 역할: 주별 집계 (데이터입력 시트 참조 수식)

  시트 4: '월별현황'
    - 역할: 월별 집계 (데이터입력 시트 참조 수식)

raw 데이터 예상 스키마:
  - 날짜: date 타입
  - 라인명: string (예: "코팅1라인", "조립2라인")
  - 생산수량: integer
  - 불량수량: integer
  - 불량유형: string (예: "외관", "치수", "기능")

[Phase 3] Mapping Strategy (매핑 전략 수립)
═══════════════════════════════════════════════════

컬럼 매핑 테이블:
  raw_data.날짜       → 양식.데이터입력.B열
  raw_data.라인명     → 양식.데이터입력.C열
  raw_data.생산수량   → 양식.데이터입력.D열
  raw_data.불량수량   → 양식.데이터입력.E열
  raw_data.불량유형   → 양식.데이터입력.F열

제외 컬럼: 없음 (모든 컬럼 사용)

데이터 타입 변환:
  - 날짜: 원본 형식 유지 (Excel date serial)
  - 수량: integer 유지
  - 문자열: 그대로

삽입 전략:
  - 방식: 클리어 후 삽입 (기존 데이터 완전 교체)
  - 시작 위치: B3 (헤더 B1:F1은 보존)
  - 방향: 행 방향 (위→아래)

[Phase 4] Execution Planning (실행 계획 수립)
═══════════════════════════════════════════════════

14개 step으로 구성:

Step 1-2: 파일 열기 (raw, template)
Step 3: raw 데이터 읽기 → $raw_data에 저장
Step 4: 데이터 검증 (필수 컬럼 존재 확인)
Step 5: 양식 기존 데이터 클리어 (B3:F10000)
Step 6: 데이터 쓰기 (raw → 양식, 컬럼 매핑 적용)
Step 7: 수식 재계산 트리거
Step 8: 양식 저장 (결과 파일)
Step 9: 결과 파일 재오픈 (data_only=true, 수식 결과 읽기)
Step 10: 일별현황 결과 읽기 → $daily_summary
Step 11: 불량률 5% 초과 검증 → $high_defect_alert
Step 12: 메시지 포맷팅 → $messenger_message
Step 13: 메신저 발송 (require_confirm=true)
Step 14: 완료 로그

데이터 흐름:
  Step 3 → $raw_data → Step 4, 6
  Step 10 → $daily_summary → Step 11, 12
  Step 12 → $messenger_message → Step 13

on_fail 전략:
  - 핵심 step (데이터 읽기/쓰기): STOP
  - RECALCULATE: WARN_AND_CONTINUE (xlwings로 실제 재계산 가능)
  - 경고 검증: WARN_AND_CONTINUE (알림 목적)
  - 완료 로그: SKIP (실패해도 작업 자체는 완료)

═══════════════════════════════════════════════════
[Phase 5] Validation Design (검증 설계)
═══════════════════════════════════════════════════

검증 규칙:
  1. raw 데이터 행 수 > 0 (빈 데이터 방지)
  2. 필수 컬럼 5개 모두 존재
  3. 생산수량, 불량수량 ≥ 0 (음수 방지)
  4. 불량률 0~100% 범위 (sanity check)
  5. 불량률 > 5% 라인 경고

이상치 탐지:
  - 불량률 5% 초과 시 경고 메시지 생성
  - 불량수량 > 생산수량인 행이 있으면 ERROR

재사용 판단:
  이 계획은 동일 양식(defect_rate_template.xlsx) + 동일 프로세스에 대해
  재사용 가능하다. template_hash가 변경되지 않는 한 매일 동일 plan으로
  실행할 수 있다. (autooffice cache run daily_defect_rate_report --data ./today/)
